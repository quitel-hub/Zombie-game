# 🧟 Zombie Survival Game

![C++](https://devdocs.io/cpp/)
![SFML](https://www.sfml-dev.org/documentation/3.0.2/)

## ✨ Особливості

### ⚔️ Бойова система
* **Різноманітний арсенал:**
    * `Gun`: Вогнепальна зброя для дистанційного бою.
    * `Sword`: Зброя ближнього бою для економії набоїв.
* **Типи ворогів:**
    * **Зомбі:** Повільні, але небезпечні у групах.
    * **Боси (`Boss.h`):** Унікальні супротивники з підвищеним запасом здоров'я та особливою поведінкою.

### 🎒 Ігровий процес
* **Система інвентарю:** Реалізована через `Inventory.h` та `Container.h`. Гравці можуть підбирати предмети, зброю та аптечки.
* **Процедурна генерація:** Елементи мапи (`Map.h`) створюють унікальний досвід при кожному запуску.
* **Звуковий супровід:** Використання OpenAL (через SFML Audio) для просторових звуків пострілів та оточення.

## 🛠 Технологічний стек

* **Мова програмування:** C++ (C++17 або новіше).
* **Графічний рушій:** [SFML 2.6.1](https://www.sfml-dev.org/) (модулі: Graphics, Window, System, Audio, Network).
* **Система збірки:** CMake (мінімальна версія 3.10).
* **Робота з даними:** JSON (бібліотека `nlohmann/json`) для конфігурацій та збережень.
* **Документація:** Doxygen (для генерації сторінок GitHub Pages).

## 🚀 Встановлення та запуск
### Вимоги
* Встановлений компілятор C++ (GCC, Clang або MSVC).
* CMake.
* Git.

### Крок 1: Клонування
* git clone [https://github.com/quitel-hub/zombie-game.git](https://github.com/quitel-hub/zombie-game.git)
* cd zombie-game

### Крок 2: Збірка (Windows/Linux/macOS)
* mkdir build
* cd build
* cmake ..
* cmake --build . --config Releasee

### Крок 3: Запуск
Після успішної збірки виконуваний файл з'явиться у папці bin (або build/Debug на Windows).
* **Важливо:** Переконайтеся, що папка assets знаходиться поруч із .exe файлом.

## 🎮 Керування
Керування реалізовано через патерн Command, що дозволяє гнучко перепризначати клавіші.
|Дія|Клавіша|Опис|
|Рух|"W, A, S, D"|Переміщення персонажа|
|Вогонь / Атака|"F"|Використання поточної зброї|
|Зміна зброї|"Q"|Перемикання слотів|
|Перезарядка|"R"|Перезарядка зброї| 
|Меню|"ESC"|Пауза / Меню

## 🏗 Архітектура та патерни
У проєкті використано кілька ключових патернів проєктування для забезпечення чистоти коду:

* **Entity Component System (спрощена):** Клас Entity є базовим для всіх живих об'єктів (Player, Zombie, Boss). Це дозволяє уніфікувати обробку фізики та колізій.
* **Command Pattern (Command.h):** Відділяє логіку вводу від логіки виконання дій. Це дозволяє легко додавати підтримку джойстиків або змінювати розкладку клавіатури.
* **Singleton (LocalizationManager.h):** Гарантує існування лише одного екземпляра менеджера локалізації, надаючи глобальну точку доступу до текстових ресурсів (en.json, ukr.json).
* **Observer (потенційно):** Використовується для системи досягнень або оновлення UI при зміні стану гравця (здоров'я, набої).

## 🌍 Локалізація
Гра підтримує динамічну зміну мови. Тексти зберігаються у JSON файлах у папці assets/:
* en.json — Англійська мова (за замовчуванням).
* ukr.json — Українська мова.

Щоб додати нову мову, просто створіть новий .json файл за зразком і зареєструйте його в LocalizationManager.

## 🧪 Тестування
Для перевірки стабільності гри використовуються Unit-тести, розташовані в папці tests/. Вони перевіряють:
* Коректність нанесення шкоди (GameLogicTests).
* Роботу інвентарю.
* Завантаження мапи.

* **Запуск тестів:**
```text
cd build
ctest -V
```
